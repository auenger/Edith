# Rollout Plan

## Stage 0 — Clarify
- Goal: define why EDITH exists and what loop to unlock first.
- Exit criteria:
  - [ ] business intent is explicit
  - [ ] first valuable loop is chosen
  - [ ] initial owners are identified

## Stage 1 — Pilot
- Goal: inventory the minimum sources, repos, and workflows needed for the first loop.
- Exit criteria:
  - [ ] source inventory exists
  - [ ] repo inventory exists
  - [ ] workflow inventory exists
  - [ ] first backlog exists

## Stage 2 — Operationalize
- Goal: make the pilot loop usable by more than one owner or one agent.
- Exit criteria:
  - [ ] key source / repo / workflow skills are drafted
  - [ ] maintenance rules exist
  - [ ] ownership and handoff are explicit

## Stage 3 — Scale
- Goal: expand to adjacent loops and reduce rediscovery cost across teams.
- Exit criteria:
  - [ ] adjacent workflows are prioritized
  - [ ] skill coverage expands intentionally
  - [ ] governance and update triggers are working
