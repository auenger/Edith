# Rollout 计划

## Stage 0 — Clarify
- Goal: 定义 EDITH 为什么存在，以及首先要解锁哪条闭环。
- Exit criteria:
  - [ ] business intent 已明确
  - [ ] 第一条有价值闭环已选定
  - [ ] 初始 owners 已识别

## Stage 1 — Pilot
- Goal: 盘点第一条闭环所需的最小 sources、repos 和 workflows。
- Exit criteria:
  - [ ] source inventory 已存在
  - [ ] repo inventory 已存在
  - [ ] workflow inventory 已存在
  - [ ] 第一版 backlog 已存在

## Stage 2 — Operationalize
- Goal: 让 试点 闭环可被不止一个 owner 或一个 agent 使用。
- Exit criteria:
  - [ ] 关键 source / repo / workflow skills 已起草
  - [ ] maintenance rules 已存在
  - [ ] ownership 与 handoff 已明确

## Stage 3 — Scale
- Goal: 扩展到相邻闭环，并降低团队间的重新发现成本。
- Exit criteria:
  - [ ] 相邻 workflows 已被排定优先级
  - [ ] skill coverage 在有意识地扩展
  - [ ] governance 与 update triggers 正在生效
