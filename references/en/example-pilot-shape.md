# Example Pilot Shape

Use this only as a shape, not as a literal company blueprint.

## Pilot goal

Prove that agents can help complete a recurring cross-repo loop with less rediscovery and less waiting.

## Example scope

### Included sources
- product docs
- issue tracker
- release notes

### Included repos
- frontend repo
- backend repo
- docs repo
- test or automation repo

### Included workflow
- bug intake → triage → fix → regression → release note

## Example outputs

- `jarvis-build-brief.md`
- `source-inventory.md`
- `repo-inventory.md`
- `workflow-inventory.md`
- `skill-backlog.md`
- `ownership-map.md`
- `rollout-plan.md`
- 1 source skill stub
- 2 repo skill stubs
- 1 workflow skill stub

## What this pilot should prove

- agents can find the right historical context faster
- agents can route into the right repos more reliably
- agents can follow a real closed loop rather than stopping at a local answer
- humans can continue the JARVIS rollout instead of treating it as a one-off artifact

## What this pilot does not need to prove

- full-company coverage
- every source integrated
- every repo skill finished
- perfect governance

The first pilot should prove direction, not exhaustiveness.
