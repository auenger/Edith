# 写入契约

生成出来的 EDITH 产物，应该对“什么内容属于哪里”保持明确主张。

## 全局契约

1. **索引，不要倾倒。** 优先写摘要、模式和路由线索，而不是复制原始内容。
2. **每条事实只保留一个归属。** 路由到最佳权威来源，而不是制造许多半真半假的副本。
3. **把方法和公司事实分开。** 可复用说明与公司专属事实必须可区分。
4. **优先保留可持续知识。** 记录能帮助未来 agents 和 owners 的知识，而不是短暂闲聊。
5. **为继续推进而写。** 另一个 agent 或 owner 应能无需重新发现基础信息就继续工作。

## 产物契约

### Build brief
应包含：
- business intent
- scope
- first valuable loop
- assumptions
- risks
- next step

不应包含：
- 伪装出的“已经定稿”感
- 对整个公司的穷尽式发现

### Source inventory
应包含：
- important sources
- owners
- access paths
- candidate source skills
- relevance and priority

不应包含：
- 复制来的原始 source 内容

### Repo inventory
应包含：
- repo role
- owners
- 入口点
- current 权威来源 locations
- candidate repo skills

不应包含：
- 应该属于 repo skill 的深度 repo-local procedures

### Workflow inventory
应包含：
- triggers
- repos and teams involved
- 产物
- success 证据
- candidate workflow skills

不应包含：
- 为每个相关 repo 编写的逐步 repo-local instructions

### Skill backlog
应包含：
- layer
- target
- owner
- priority
- readiness
- notes

不应包含：
- 没有理由的模糊“someday”条目

### MAINTENANCE guide
应包含：
- mission
- scope
- access paths
- write contracts
- update triggers
- ownership
- closed-loop maintenance rules

不应包含：
- 对继续推进没有影响的低信号操作碎事
