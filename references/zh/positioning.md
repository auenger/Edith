# 定位

## JARVIS 是什么

JARVIS 是一个**agent-first 的组织能力层**。

它帮助 agents：
- 理解公司的数字资产，
- 路由到正确的 repos 和执行面，
- 跟随真实 workflows，
- 并保留它们学到的东西，使组织能力随着时间推移产生复利。

一次 JARVIS rollout 通常包括：
- memory 与 routing，
- 面向 source 的 skills，
- repo-local skills，
- workflow skills，
- rollout ownership，
- 以及更新纪律。

## JARVIS 不是什么

JARVIS 不是：
- 原始文档镜像，
- issue 倾倒仓，
- 一次性 repo generator，
- repo-local 权威来源 的替代品，
- 也不是某位专家一次性就能做完全部工作的承诺。

## 真正的价值单位

真正的价值单位不是“一个生成好的 repo”。

真正的价值单位是一个**可运行的闭环**，在这个闭环里，agent 能够：
1. 找到正确的公司上下文，
2. 进入正确的 repo 或系统，
3. 帮助完成真实工作，
4. 并把可持续复用的学习写回去。

如果 rollout 无法解锁这样的闭环，那它仍然只是 scaffold。

## 为什么这很重要

没有 JARVIS，agents 只能停留在会话局部，并且失忆。

有了 JARVIS，组织就可以开始：
- 降低跨团队等待成本，
- 保留工程判断，
- 让更多角色参与更完整的交付闭环，
- 并逐步走向 One Person Team 的工作模式。

## 工作姿态

把 JARVIS 建设视为：
- 组织设计，
- 知识架构，
- workflow enablement。

不要把它当成一次文档美化练习。
