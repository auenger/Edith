# 压缩规则

源文本压缩为蒸馏格式的最终处理规则。

## 完全删除

- 过渡语："如前所述"、"值得注意的是"、"此外"
- 修辞和说服："这是颠覆性的"、"令人兴奋的是"
- 犹豫和模糊："我们认为"、"可能"、"似乎"、"或许"
- 自指："本文描述了"、"如上所述"、"如下所示"
- 常识解释："JSON 是一种数据交换格式"、"Git 是版本控制工具"
- 重复引入同一概念
- 段落过渡句
- 纯装饰元素（加粗强调、分割线）
- 填充短语："为了"、"需要注意的是"、"事实上"

## 必须保留

- 具体数字、日期、版本、百分比
- 命名实体（产品、公司、人员、技术）
- 决策及理由（压缩为："决策：X。理由：Y"）
- 被拒方案及原因（压缩为："被拒：X。原因：Y"）
- 明确约束和不可妥协项
- 依赖和顺序关系
- 未决问题和待办事项
- 范围边界（包含/排除/延期）
- 成功标准及验证方法
- 用户角色和各自的定义
- 风险及严重程度
- 源文档间的冲突

## 转换规则

| 原始形式 | 压缩形式 |
|---------|---------|
| 散文段落 | 单条密集信息点 |
| "我们决定用 X 因为 Y 和 Z" | "X（理由：Y, Z）" |
| 重复分类标签 | 合并到同一标题下 |
| "风险：... 严重程度：高" | "HIGH RISK: ..." |
| 条件语句 | "如果 X → Y" |
| 多句解释 | 分号分隔压缩形式 |
| 相关短项列表 | 单条信息 + 括号列表 |
| "X 用于 Y" | "X: Y"（上下文明确时） |
| 详细列举 | 括号："平台（Cursor, Claude Code, Windsurf）" |

## 去重规则

- 同一事实在多个文档中 → 保留上下文最丰富的版本
- 同一概念不同详细程度 → 保留详细版
- 重叠列表 → 合并去重
- 源文档冲突 → 明确标注："文档 A 说 X；文档 B 说 Y — 未解决"
- 执行摘要在别处有详细展开 → 只保留展开版
- 跨段落重复的引导语 → 在最相关主题下记录一次
